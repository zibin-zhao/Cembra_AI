================================================================================
ARCHITECTURE OVERVIEW
================================================================================

MODEL PIPELINE:
  Input: SNP Features [batch, max_snps, input_dim]
    ↓
  SNPFeatureEncoder (+ positional encoding)
    ↓
  LDBlockTransformer (multi-head attention per block)
    ↓
  Block Pooling (mean aggregation)
    ↓
  CrossBlockAttention (sparse top-k)
    ↓
  Global Representation [batch, d_model]
    ↓
  ┌─────────────────────────┬──────────────────────┐
  RiskPredictionHead        DomainDiscriminator
  ↓                         ↓
  Risk Logits [batch]       Domain Logits [batch]

TRAINING PHASES:
  Phase 1: EUR Pre-training (20 epochs)
    - Objective: Learn risk prediction
    - Loss: Binary cross-entropy
    - Data: EUR ancestry only
  
  Phase 2: Domain Adaptation (30 epochs)
    - Objective: Generalize across ancestries
    - Loss: EUR risk + EAS risk + adversarial domain
    - Data: Multi-ancestry (EUR + EAS)
    - Lambda: Annealed 0→1
  
  Phase 3: Fine-tuning (10 epochs)
    - Objective: Adapt to target data
    - Frozen: Backbone (encoder, transformer)
    - Trainable: Risk head only
    - Data: Individual/mixed ancestry

================================================================================
INPUT/OUTPUT SPECIFICATIONS
================================================================================

FORWARD PASS INPUTS:
  snp_features:      [batch, max_snps, input_dim]  - per-SNP features
  block_indices:     [batch, max_snps]             - LD block assignment
  block_masks:       [batch, max_snps]             - SNP validity (bool)
  ancestry_labels:   [batch]                       - optional for domain loss
  
OUTPUT DICTIONARY:
  risk_logits:           [batch, 1]                - risk predictions
  domain_logits:         [batch, 1]                - ancestry predictions
  block_representations: [batch, n_blocks, d_model] - block features
  global_representation: [batch, d_model]          - global features
  attention_weights:     [batch, n_heads, ...]     - optional
